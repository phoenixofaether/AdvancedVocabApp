# AdvancedVocabApp

Vocabulary learning app for advanced English learners (Oxford Advanced to Proficiency level). Full-stack monorepo: React 19 frontend, .NET 10 backend, shared TypeScript package.

## Architecture

**Monorepo layout:**
```
apps/web/          - React 19 SPA (Vite, TanStack Router, TanStack Query, Zustand, Tailwind 4)
packages/shared/   - @vocabapp/shared: Zod schemas, TS types, constants (source-linked, no build step)
server/src/
  AdvancedVocabApp.Api/            - ASP.NET Core controllers, Program.cs, DI wiring
  AdvancedVocabApp.Core/           - Entities, DTOs, Interfaces (no outward dependencies)
  AdvancedVocabApp.Infrastructure/ - AppDbContext (EF Core + Pomelo MySQL), Services
  AdvancedVocabApp.Tests/          - xUnit tests
```

**Clean architecture rule:** Core has zero dependencies on Infrastructure or Api. Infrastructure depends on Core. Api depends on both.

**Frontend data flow:** `apiClient` (apps/web/src/api/client.ts) is an Axios instance that proxies `/api/*` via Vite to `:5000`. JWT access token read from `localStorage.accessToken`; 401 triggers silent refresh via `POST /api/auth/refresh` and queues pending requests. On refresh failure, redirects to `/login`.

**Auth:** Google OAuth only (no password login). Backend validates Google ID tokens via `Google.Apis.Auth`, issues JWT access tokens + refresh tokens (rotated on use). Tokens stored in `localStorage`.

## Commands

### Frontend (run from repo root)
```bash
pnpm dev:web          # Start web dev server on :5173
pnpm build:web        # Build web app only
pnpm build:all        # Build shared then web (for production)
pnpm typecheck        # Type-check all packages
pnpm lint             # Lint all packages
```

> Note: `pnpm build:shared` is defined in root scripts but the shared package has no `build` script (it's source-linked). Use `pnpm build:all` for production builds.

### Backend
```bash
# From server/src/AdvancedVocabApp.Api/
dotnet run            # Start API on :5000, Swagger at /swagger
dotnet watch run      # Hot reload

# From repo root
dotnet test server/src/AdvancedVocabApp.Tests/
```

### EF Core Migrations
```bash
# Run from repo root
dotnet ef migrations add <Name> \
  --project server/src/AdvancedVocabApp.Infrastructure \
  --startup-project server/src/AdvancedVocabApp.Api

dotnet ef database update \
  --project server/src/AdvancedVocabApp.Infrastructure \
  --startup-project server/src/AdvancedVocabApp.Api
```

**Requirements:** Node ≥22, pnpm ≥10, .NET 10 SDK, MySQL running locally.

## Key Conventions and Gotchas

**NEVER edit `apps/web/src/routeTree.gen.ts`** — auto-generated by TanStack Router Vite plugin on every dev/build run.

**Secret files are gitignored — never commit them:**
- `server/src/AdvancedVocabApp.Api/appsettings.Development.json` — DB connection string, JWT key, Google Client ID
- `server/src/AdvancedVocabApp.Api/appsettings.Production.json`
- All `.env*` files

**`appsettings.json`** is committed and contains placeholder values only (e.g. `"CHANGE_ME"`). Real secrets go in `appsettings.Development.json`.

**Shared package is source-linked.** `@vocabapp/shared` exports TypeScript source directly (`"main": "./src/index.ts"`). No build step needed during development.

**When changing an API contract:** Update both C# DTO records (`server/src/AdvancedVocabApp.Core/DTOs/`) and the mirrored TS interfaces (`packages/shared/src/types/index.ts`).

**Word normalization:** Words are stored `word.Trim().ToLowerInvariant()` in `VocabEntriesController`. Apply the same when doing lookups.

**CORS:** Origins come from `Cors:AllowedOrigins` in config (set in `appsettings.Development.json`).

**Routing pattern:** `_authenticated.tsx` layout route guards all child routes by checking `localStorage.accessToken`. Authenticated routes live under `apps/web/src/routes/_authenticated/`.

**Pending features (interfaces defined, not yet implemented):**
- `IDictionaryService` — interface in Core, no implementation or DI registration yet
- `ITextToSpeechService` — same
- `ISpacedRepetitionService` is registered in DI (`SpacedRepetitionService` in Infrastructure) but no controller uses it yet — spaced repetition review flow is pending

## API Routes

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| POST | /api/auth/google | No | Google login — validates ID token, returns JWT + refresh token |
| POST | /api/auth/refresh | No | Rotate refresh token, return new JWT + refresh token |
| POST | /api/auth/logout | Yes | Revoke refresh token |
| GET | /api/auth/me | Yes | Current user profile |
| GET | /api/vocab-sets | Yes | List all sets for current user |
| POST | /api/vocab-sets | Yes | Create new vocab set |
| GET | /api/vocab-sets/{id} | Yes | Get set by ID (includes entry count) |
| PUT | /api/vocab-sets/{id} | Yes | Update set name/description |
| DELETE | /api/vocab-sets/{id} | Yes | Delete set (cascades entries) |
| POST | /api/vocab-sets/{id}/entries | Yes | Add existing entries to a set |
| DELETE | /api/vocab-sets/{id}/entries/{entryId} | Yes | Remove entry from set |
| GET | /api/vocab-entries/{id} | Yes | Get entry by ID (includes dictionary data) |
| POST | /api/vocab-entries | Yes | Create entry + auto-create ReviewCard |
| PUT | /api/vocab-entries/{id} | Yes | Update custom definition/phonetic only |
| DELETE | /api/vocab-entries/{id} | Yes | Delete entry |

## Critical Files

| File | Purpose |
|------|---------|
| `server/src/AdvancedVocabApp.Api/Program.cs` | DI registration, middleware pipeline — register new services here |
| `server/src/AdvancedVocabApp.Core/DTOs/` | C# request/response records — pattern for new endpoints |
| `server/src/AdvancedVocabApp.Core/Entities/` | EF Core entity classes |
| `server/src/AdvancedVocabApp.Infrastructure/Data/AppDbContext.cs` | EF Core config — new entities need DbSet + OnModelCreating entry |
| `server/src/AdvancedVocabApp.Infrastructure/Services/SpacedRepetitionService.cs` | SM-2 implementation |
| `packages/shared/src/types/index.ts` | Canonical frontend TS interfaces — keep in sync with C# DTOs |
| `apps/web/src/api/client.ts` | Axios instance with auth interceptors — pattern for all frontend API calls |
| `apps/web/src/api/auth.ts` | TanStack Query hooks for auth |
